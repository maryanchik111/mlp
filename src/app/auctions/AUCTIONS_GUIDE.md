# icons Система Аукціонів

## Опис

Система аукціонів дозволяє адміністраторам створювати аукціони на товари, а користувачам брати участь у ставках за рідкісні предмети.

## Основні функції

### Для адміністраторів:

1. **Створення аукціону** (в адмін-панелі → Аукціони):
   - Назва та опис товару
   - Стартова ціна
   - Крок ставки (мінімальний приріст, наприклад 50₴)
   - Час неперебивання ставки (таймаут, мін)
   - Час відкриття аукціону (точний час, коли аукціон стане активним)
   - URL зображення товару

2. **Управління**:
   - Перегляд всіх аукціонів (активні, запланові, завершені)
   - Видалення аукціонів
   - Автоматичне закриття при закінченні таймауту

### Для користувачів:

1. **Перегляд аукціонів** (сторінка `/auctions`):
   - **Активні** - аукціони, в яких можна робити ставки
   - **Запланові** - аукціони, які будуть відкриті пізніше
   - **Архів** - завершені аукціони

2. **Участь у ставках** (тільки авторизовані):
   - Введення суми ставки (не менше поточної ціни + крок)
   - Автоматичне оновлення таймеру
   - Перегляд історії ставок

## Логіка роботи

### Цикл життя аукціону:

1. **Scheduled (Запланований)**
   - Час до відкриття відображається
   - Ставки неможливі
   - В точний час переходить в `active`

2. **Active (Активний)**
   - Користувачі можуть робити ставки
   - Таймер відраховує залишилось часу з моменту останньої ставки
   - При кожній новій ставці таймер скидається
   - Якщо таймер дійде до 0 - аукціон закривається

3. **Closed (Завершений)**
   - Переглядання результатів
   - Вказано переможця та фінальну ціну
   - Ставки неможливі

### Таймер неперебивання:

- Таймер починається з моменту **останньої ставки**
- За будь-яку нову ставку таймер **скидається повністю**
- Якщо час вичерпав - аукціон автоматично закривається
- Переможцем стає той, хто зробив останню ставку

## Приклад використання

### Адміністратор створює аукціон:

```
Назва: "Рідка фігурка Rainbow Dash Gold Edition"
Стартова ціна: 500₴
Крок ставки: 50₴
Таймаут: 30 хв
Відкриття: 20:00 (сьогодні)
```

### Користувач робить ставки:

- 20:00 - Аукціон відкривається
- 20:05 - Користувач А ставить 550₴ (таймер: 30 хв)
- 20:10 - Користувач Б ставить 600₴ (таймер: 30 хв, з цього моменту)
- 20:35 - Таймер закінчується (30 хв з останньої ставки)
- 20:35 - Аукціон закивається, переможець: Користувач Б (600₴)

## API Endpoints

### Перевірка та оновлення аукціонів
```
GET /api/auctions/check
```
Перевіряє запланові та активні аукціони, переводить їх у відповідні стани.

## Важливо

- Аукціони не можна редагувати після створення - видаліть та створіть новий
- Таймер наявний тільки на клієнті - реальне закриття відбувається на сервері
- Рекомендується перекликати API endpoint для синхронізації статусів
- Авторизація обов'язкова для участі у ставках
