# MLP E-commerce - Зміни та Інструкції

## Що змінено

### 1. Додано кількість товарів
- Кожен товар тепер має поле `quantity` в базі даних
- При покупці кількість автоматично зменшується
- Коли кількість = 0, товар стає недоступним (`inStock = false`)

### 2. Людські номери замовлень
- Замість Firebase push ID, тепер використовуються зрозумілі номери типу **NW4343**
- Формат: 2 великі літери + 4 цифри (наприклад: AB1234, XY9876)

### 3. Адмін панель - нова вкладка "Товари"
- Додано вкладку "Товари" поряд з "Замовлення"
- Показує всі товари з інформацією:
  - Назва, категорія, ціна
  - Кількість на складі
  - Опис
  - Статус наявності
- Кнопка "Редагувати" дозволяє змінити:
  - Назву
  - Категорію
  - Ціну
  - Кількість
  - Опис
  - Іконку (emoji)

### 4. Каталог тепер працює з базою даних
- Товари завантажуються тільки з Firebase
- Видалено fallback товари
- Додано стан завантаження
- Додано обробку порожньої бази

## Як налаштувати

### Крок 1: Налаштування Firebase

1. Створіть проект у [Firebase Console](https://console.firebase.google.com/)
2. Увімкніть Realtime Database
3. Скопіюйте конфігурацію у `.env.local`:

```env
NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_DATABASE_URL=https://your_project.firebaseio.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id
```

### Крок 2: Завантажити товари в базу

Запустіть скрипт для завантаження товарів з `products.json` у Firebase:

```bash
node scripts/upload-products.js
```

Цей скрипт:
- Читає `public/products.json`
- Завантажує всі товари в Firebase Realtime Database
- Виводить статус завантаження

### Крок 3: Запустіть проект

```bash
npm run dev
```

## Використання

### Для адміністратора

1. Відкрийте `/admin`
2. Перейдіть на вкладку "Товари"
3. Натисніть "Редагувати" на будь-якому товарі
4. Змініть потрібні поля
5. Натисніть "Зберегти зміни"

### Робочий процес замовлень

1. Клієнт додає товари в кошик
2. Оформлює замовлення
3. Система:
   - Створює замовлення з людським номером (напр. NW4343)
   - Зменшує кількість товарів у базі
   - Перенаправляє на оплату
4. Адміністратор:
   - Бачить нове замовлення в панелі
   - Підтверджує оплату
   - Виконує замовлення

## Важливі зауваження

1. **Безпека**: Додайте правила безпеки Firebase для захисту даних
2. **Резервне копіювання**: Регулярно робіть бекап бази даних
3. **Індексація**: Створіть індекси для швидкого пошуку замовлень
4. **Моніторинг**: Слідкуйте за кількістю товарів та вчасно поповнюйте склад

## Troubleshooting

**Товари не завантажуються?**
- Перевірте налаштування Firebase у `.env.local`
- Запустіть скрипт `upload-products.js`
- Перевірте правила безпеки Firebase

**Помилка при зміні товару?**
- Переконайтеся, що ви залогінені як адмін
- Перевірте підключення до Firebase

**Замовлення не створюється?**
- Перевірте консоль браузера на помилки
- Переконайтеся, що товари в наявності
